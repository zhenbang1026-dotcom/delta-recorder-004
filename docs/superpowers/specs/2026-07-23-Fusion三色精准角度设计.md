# Fusion 三色精准角度识别设计

## 目标

在不修改 Legacy 和现有 TEXT 识别行为的前提下，把 `D:\Azhuomian\GitHub\text\识别角度.py` 中绿色、蓝色、黄色的 HSV 识别能力及各自标定数据接入 Fusion。Fusion 使用三色精准识别作为主观测，Legacy 继续负责同帧交叉验证和失败降级，程序默认模式仍为 Legacy。

这里的“引进”是功能性移植，不直接复制外部脚本。外部脚本的 `dxcam`、实时循环、调试窗口和固定 `+90°` 输出与 004 的统一截图及 MAP 插值不兼容，不进入运行链路。

## 已核实的基线

- 当前分支：`feature/text-fusion-smooth`，基线提交 `b3cc60d`。
- 基线测试：51 项通过。
- 当前项目 `校准截图/lv.txt` 与外部绿色 `校准截图绿色/lv.txt` 的 SHA256 完全相同。
- 外部最新版支持按参数手动选择三色，不会自动判断当前颜色。
- 外部脚本读取 `MAP:`，但最终仍返回固定 `(原始角度 + 90) % 360`；004 现有 TEXT 已实现 MAP 圆周残差插值，必须保留。
- 三个“大颜色”目录的 `lv.txt` 是运行时标定数据；三个“小颜色”目录只有生成过程中的裁剪图，运行时不依赖。

## 范围与非目标

### 本次范围

- 新增只供 Fusion 使用的三色精准识别器。
- 复制三份 Fusion 专用 `lv.txt`，不依赖工作区外的绝对路径。
- 自动检测绿、蓝、黄，并对当前颜色做短时锁定。
- 为候选结果增加几何质量门禁。
- 把精准结果接入现有 Fusion 主观测位置，继续使用 Legacy 校验与降级。
- 增加自动化测试、诊断字段和中文文档。

### 明确不做

- 不修改 Legacy 的识别、ROI、颜色遍历、控制或在线标定。
- 不修改现有 TEXT 的识别器、MAP 插值、稳定器或模式分发。
- 不修改截图模块、路线策略、连续视角控制和 Win32 输入。
- 不增加颜色选择 UI；Fusion 自动判断颜色。
- 不复制 1079 张 BMP 校准图，也不把三个“小颜色”目录放进项目。
- 不引入多线程图像识别；ROI 很小，优先保证确定性，颜色锁定后每帧只运行一个 profile。

## 方案比较与决定

### 方案 A：Fusion 精准三色主观测 + Legacy 降级（采用）

现有 TEXT 保留为独立对照模式。Fusion 的主观测换成精准三色识别，Legacy 继续同帧校验。隔离边界清晰，满足用户指定的 Legacy/TEXT 不变。

### 方案 B：精准三色 + TEXT + Legacy 三路融合（不采用）

精准绿色与 TEXT 使用同类 HSV、连通域和 MAP 数据，并非独立传感器。增加第三路会重复计算，却不能提供等价的独立证据。

### 方案 C：新增第四种模式（不采用）

会扩大 UI、配置和巡航分支，用户已明确选择把能力加到新的 Fusion 算法中。

## 组件与数据流

新增 `三色精准角度.py`，包含冻结的颜色 profile、单色识别器实例、几何质量计算以及颜色锁定状态。它复用 004 已验证的角度识别和 MAP 插值能力，不创建自己的截图后端。

数据流：

```text
统一截图模块给出的 193×193 雷达 BGR 图
  ├─ Fusion 三色精准识别器
  │    ├─ 当前锁定色单色识别
  │    ├─ 失败时探测另外两色
  │    ├─ 几何质量门禁
  │    └─ 对应颜色 MAP 插值 → 精准主观测
  └─ 原 _analyze_image_legacy（同帧小 ROI，源码不改）
          ↓
     原角度融合选择器
          ↓
     现有连续视角控制
```

`A测试角度识别.py` 只在 `_analyze_image_fusion` 中惰性取得三色识别器。Legacy 和 TEXT 的分发函数及识别函数保持源码快照不变。进入 Fusion 时重置颜色状态；进入 Legacy/TEXT 不会加载三色资产。

## 自动颜色状态机

### 未锁定

- 检测三个 profile。
- 候选必须同时通过原有连通域要求和新增几何门禁。
- 选择质量分最高的候选并锁定；完全无候选时精准主观测失败，由 Fusion 使用 Legacy。

### 已锁定

- 正常帧只检测当前颜色，成功后清除失败计数和待切换候选。
- 当前颜色失败时才检测另外两色。
- 同一个新颜色连续 2 帧通过门禁后切换；确认期间精准主观测返回失败，由 Fusion 暂时使用 Legacy。
- 当前颜色在确认期间恢复时，立即取消待切换状态。
- 不保留陈旧角度，不用上一颜色的 MAP 解释新颜色。

连续两帧约等于当前 50ms 识别节拍下的 100ms，目的是挡住单帧同色 HUD 干扰，同时避免明显切色停顿。

## 几何质量门禁

外部全量校准样本显示：圆盘中心偏差小于 0.7px，箭头半径约 15.1–16.4px，mask 像素约 369–459。实机画面会有压缩、亮度和缩放波动，因此生产门禁使用宽松边界：

- 圆盘中心相对标定中心偏差不大于 5px。
- 箭头中心半径在 10–22px。
- mask 非零像素在 200–800。
- 至少存在两个非背景连通域。
- MAP 插值结果和数值必须有限且位于 `[0,360)`。

质量分由中心偏差、半径偏差、mask 偏差和 MAP 密度置信度组成，只用于未锁定时选择候选；锁定色有效时不因其他颜色理论分数更高而切换。

## Fusion 选择与诊断

- 精准观测有效、Legacy 有效且角差在一致阈值内：采用精准观测。
- 两者明显分歧：继续使用现有圆周连续性与恢复迟滞规则。
- 精准无效：降级 Legacy。
- Legacy 无效：采用精准观测。
- 两者都无效：沿用现有错误路径，不输出陈旧角度。

现有选择器内部仍可把第一个参数视为主观测；对外诊断将来源标记为 `precise`，并记录 `precise_color`、质量分、切色状态、精准错误和与 Legacy 的圆周角差，避免误显示为 TEXT。

## 错误处理

- 单个 `lv.txt` 缺失或损坏：禁用该 profile，错误包含颜色和完整路径，其他颜色继续可用。
- 三份标定都不可用：只有 Fusion 精准主观测失败；程序导入、Legacy 和 TEXT 仍可正常工作。
- 图像为空、裁剪为空、连通域不足或几何异常：本帧候选失败，不抛出跨线程未处理异常。
- 切色确认期间：不输出未确认的新角度，由 Legacy 接管。
- 所有新对象惰性创建，避免默认 Legacy 启动时依赖新资产。

## 测试设计

坚持 TDD，先写失败测试并确认红灯，再实现最小代码。

### profile 与资产

- 三色 HSV 唯一匹配代表色。
- profile 严格绑定对应路径、crop、中心、MAP 数量和文件哈希。
- 三份标定的圆周插值覆盖 `359°/0°`，留一预测 P95 小于 1.5°、最大误差小于 2.5°。
- 缺一色只禁用一色，绝不回退其他 `lv.txt`。

### 识别与切色

- 参数化合成绿、蓝、黄雷达图，角度误差不超过 2.5°。
- 空图、单连通域、远离标定中心的同色块被拒绝。
- 错误 HSV 不跨 profile 识别。
- 锁定成功后每帧只调用当前颜色。
- 新色需要连续两帧确认；单帧干扰不切换；旧色恢复会取消切换。
- 切换确认期间 Fusion 降级 Legacy。

### 隔离回归

- Fusion 调用三色精准识别且不调用 `_analyze_image_text_raw`。
- TEXT 模式不创建三色识别器。
- Legacy、`_analyze_image_text_raw`、`_analyze_image_text` 和 TEXT 默认识别器增加源码/行为快照。
- 默认模式和所有 UI 入口仍为 Legacy。
- 现有 Fusion 选择、连续控制、巡航策略和 UI 测试全部复跑。
