# Fusion 三色精准角度设计

## 目标与边界

本轮只增强 `fusion`：用绿色、蓝色、黄色三套独立 HSV 与独立 MAP 标定生成精准主观测，再用现有 Legacy 观测做交叉验证和失败降级。

- 默认模式继续是 `legacy`。
- `_analyze_image_legacy`、`_analyze_image_text_raw`、`_analyze_image_text` 和 `识别角度.默认识别器` 的源码与行为保持不变。
- TEXT 继续使用现有单绿色标定，作为独立对照。
- 不修改 `巡航脚本.py`、`连续视角控制.py`、截图模块、鼠标模块和 UI。
- 不复制校准截图，只引入三份约 21 KB 的 `lv.txt`。

## 已确认根因

上一阶段已把 TEXT/Fusion 的鼠标输出改为固定节拍、限速和限加速度的连续控制，但 Fusion 的主观测仍调用单绿色 `_analyze_image_text_raw`。角色箭头变为蓝色或黄色时，精准观测会失效，Fusion 随即切到噪声更大的 Legacy；观测源切换及角度噪声会让连续控制反复改变目标速度方向，形成左右抖动。

因此本轮修复主观测的颜色覆盖与质量门禁，不继续调整鼠标曲线。这样能直接消除错误源，同时避免破坏已经稳定的 Legacy 和现有 TEXT 对照。

## 方案比较

- **Fusion 精准三色主观测 + Legacy 降级（采用）**：现有 TEXT 保留独立对照，隔离边界最清楚。
- **精准三色 + TEXT + Legacy 三路融合（不采用）**：精准绿色与 TEXT 使用同类 HSV、连通域和 MAP，并非独立传感器，只会重复计算。
- **新增第四种模式（不采用）**：会扩大 UI、配置和巡航分支，不符合把能力加入 Fusion 的决定。

## 组件设计

新增 `三色精准角度.py`，职责只包括 Fusion 三色精准识别：

```python
@dataclass(frozen=True)
class 颜色配置:
    名称: str
    hsv下界: tuple[int, int, int]
    hsv上界: tuple[int, int, int]
    校准路径: Path
    代表色: str

@dataclass(frozen=True)
class 精准角度结果:
    angle: float
    color: str
    confidence: float
    details: dict

class 三色精准角度识别器:
    def 识别(self, 图像) -> 精准角度结果 | None: ...
    def reset(self) -> None: ...
```

每个颜色配置独立持有一个现有 `识别角度.角度识别器`，复用已经验证的连通域、角度计算和 MAP 圆形插值，不复制外部项目的 `dxcam`、固定 `+90°`、调试窗口或自动搜索标定逻辑。

实例只在 Fusion 第一次实际识别时懒加载。`A测试角度识别.set_angle_mode()` 切入 Fusion 时继续重置现有 Fusion 选择器，并在三色识别器已创建时调用其 `reset()`；尚未创建时不触发加载。Legacy/TEXT 启动和运行不依赖三色模块及新资产。

## 几何质量门禁

仅有 HSV 命中不足以证明识别有效。下面的统计和阈值全部使用“按 profile crop 裁剪后再 2× 放大”的坐标系，不得拿原始 193×193 雷达图坐标直接比较。每个候选必须同时满足：

- 检测圆心与标定中心距离不超过 `5 px`；
- 标定中心到箭头质心的半径在 `10–22 px`；
- 放大后掩码非零像素在 `200–800`。

这些阈值覆盖真实校准样本的圆心误差 `≤0.64 px`、半径约 `15.1–16.4 px`、掩码约 `369–459`，同时给运行时抗锯齿和压缩留出余量。单连通域、远离中心的同色块、异常半径和异常面积全部拒绝。

门禁通过后按以下固定公式计算质量分：

```text
中心分 = clamp(1 - 圆盘中心偏差 / 5, 0, 1)
半径分 = clamp(1 - abs(箭头半径 - 15.75) / 6.25, 0, 1)
mask分 = clamp(1 - abs(mask像素 - profile期望值) / 300, 0, 1)
质量分 = 0.35×中心分 + 0.30×半径分 + 0.20×mask分 + 0.15×MAP置信度
```

profile 的 mask 期望值固定为绿色 438、蓝色 425、黄色 380。质量分只用于未锁定时或当前色失败后选择待切换候选；锁定色有效时不探测其他颜色。多候选同帧有效时取质量分最高者，差值小于 `1e-9` 视为同分并按绿色、蓝色、黄色顺序裁决。门禁失败直接返回无观测，不能输出低质量角度。

## 颜色锁定与切换

为兼顾流畅度和性能：

1. 未锁定颜色时评估所有可用颜色，选择几何质量最高的有效候选。
2. 已锁定时每帧先且只检测当前颜色；有效就保持，避免多色阈值边缘导致来回切换。
3. 当前颜色失效时，同帧评估其他颜色并选择质量最高的待切换候选。
4. 同一个新颜色必须连续 2 帧通过门禁：第一帧不输出未确认角度，由 Fusion 暂时使用 Legacy；第二帧确认成功时立即锁定并输出该帧精准角度，不再额外降级一帧。
5. 确认期间旧颜色恢复时，立即取消待切换状态；单帧新色或同色 HUD 干扰不会改变锁定。
6. 三色都失败时返回无精准观测，让现有 Fusion 选择器立即降级 Legacy。

`reset()` 清除颜色锁定；再次进入 Fusion 时从三色重新选择。

## Fusion 接入

只在 `A测试角度识别.py` 增加 Fusion 专用懒加载器，并替换 `_analyze_image_fusion` 的主观测：

```text
同一张 193×193 雷达截图
  ├─ 三色精准识别器 → 精准主观测
  └─ 原 _analyze_image_legacy → Legacy 交叉验证/降级
                       ↓
              原 角度融合选择器
                       ↓
              原 连续视角控制链
```

现有 `角度融合选择器` 的 `text` 参数只作为“主观测槽位”，内部迟滞逻辑和测试接口保持不变。`_analyze_image_fusion` 使用以下固定映射：

- 选择器返回内部来源 `text`：取精准三色结果对象，对外 `observation_source` 改为 `precise`。
- 选择器返回 `legacy`：取 Legacy 结果对象，对外来源仍为 `legacy`。
- 内部原因文本中的 `TEXT` 在输出前替换成 `精准三色`，不重命名选择器内部状态。

不修改 `巡航脚本.py` 的日志读取层。`precise_color`、质量、切色状态和精准失败原因写入现有 `fusion_reason`，代表色写入现有 `color_hex`，双源圆周角差继续写入 `fusion_difference`。结果对象同时附加 `precise_color`、`precise_quality`、`precise_error` 供测试或其他调用方读取。

## 错误处理

- 单色资产缺失或损坏：禁用该颜色，其他颜色继续工作；错误必须包含颜色和完整路径。
- 标定损坏判定：缺少 `SRC_CROP`、`PRECISE_CENTER` 或 `MAP`；数值不是有限数；crop 不在 193×193 内或宽高非正；精准中心不在 2× crop 范围内；MAP 少于 300 条；原始角或游戏角不在 `[0,360)`；排序后的原始角最大圆周间隙超过 10°。任一条件成立即禁用该 profile。
- 单色图像门禁失败：视为该色无观测，不复用上一帧角度。
- 精准三色均失败、Legacy 成功：输出 Legacy。
- 精准成功、Legacy 失败：输出精准角度。
- 两路均失败：沿用现有 Fusion 失败行为。
- 任何失败都不得影响 Legacy/TEXT 模式的加载与调用。

## 验收标准

- 三色配置、HSV、代表色、路径、Crop、中心、MAP 数和 SHA256 全部有测试。
- 三个代表色只能命中自己的 HSV；三份 MAP 留一误差 P95 `<1.5°`、最大 `<2.5°`。
- 绿、蓝、黄合成雷达图分别输出正确颜色和合理校准角度。
- 单连通域、偏心色块、异常半径、异常面积被拒绝。
- 颜色按绿→蓝→黄切换时需连续两帧确认且不沿用旧颜色角度；锁定有效时每帧只调用一个颜色识别器。
- 缺失某色标定时不得借用其他标定；三色失败时 Fusion 降级 Legacy。
- Legacy、TEXT 关键源码哈希不变，默认仍为 Legacy。
- 现有连续控制、路线、UI 和全量测试全部通过。
- Git 新增文件中没有 BMP、PNG、JPG。
- `compileall` 和 `git diff --check` 通过，`巡航脚本.py`、`连续视角控制.py` 与基线无差异。
