# 角度融合与连续转向设计

## 目标与不可变边界

- `Legacy` 的识别、路线抽稀、动作选择、鼠标执行和在线标定逻辑保持原样。
- 默认角度模式仍为 `legacy`。
- `TEXT` 使用现有 `MAP:` 校准数据输出游戏角度，并减少重复滤波造成的相位滞后。
- 新增 `fusion` 模式：TEXT 为主观测，Legacy 只做交叉验证与故障降级，不做无条件平均。
- 仅 TEXT/Fusion 使用更密的自动路线、近点保护和连续转向限制。

## 诊断证据

`校准截图/lv.txt` 包含 353 个 `原始实测角度 -> 游戏角度` 样本。当前代码读取了这些样本，但输出仍固定使用 `(原始角度 + 90) % 360`。

离线统计结果：

| 方法 | 平均绝对误差 | P95 绝对误差 |
|---|---:|---:|
| 固定 `+90°` | 27.983° | 29.519° |
| MAP 圆形插值（留一预测） | 0.363° | 0.882° |

标定原始角的最大相邻间隙为 4.051°，足以支持分段圆形插值。录像日志同时显示 TEXT 的平均绝对角差 24.70°、72 次转向和 50.9% 微调占比，符合“系统偏角 + 延迟补偿 + 大命令反向修正”的因果链。

## 方案比较

### 方案 A：只修正 TEXT 标定

改动最小，能消除约 28°系统误差，但无法隔离单帧识别失败，也不能解决大鼠标命令和稀疏航点带来的控制不连续。

### 方案 B：修正 TEXT，并新增 Fusion 与连续控制（采用）

先消除观测偏差，再通过双算法一致性选择、故障降级和只作用于 TEXT/Fusion 的控制整形降低抖动。Legacy 保持完整对照，风险可通过模式分流与回归测试约束。

### 方案 C：重写统一状态观察器

理论上可统一坐标、角度和鼠标运动模型，但会大范围改动现有稳定链路，难以满足 Legacy 不变，当前不采用。

## 角度数据流

### TEXT

1. HSV 与连通域提取箭头方向，得到 `[0, 360)` 原始角。
2. 将每条 `MAP:` 转换为 `游戏角 - 原始角` 的有符号圆形残差；重复原始角用圆形均值合并。
3. 将残差样本按原始角排序，找到查询角两侧样本；首尾区间按 `+360°` 展开。
4. 对残差按最短圆弧线性插值，将结果加回原始角并归一化到 `[0, 360)`。
5. 仅保留一层速度自适应滤波：小抖动轻度平滑，正常转向直接跟随；删除控制端命令角与读数的重复混合。
6. 校准文件没有有效 MAP 时保留旧公式作为兼容回退，并在详情中记录校准来源。

### Fusion

Fusion 使用 TEXT 大 ROI 一次截图，并从其中裁出 Legacy 小 ROI，避免增加截图时序差。

- 两者都成功且圆周差在一致阈值内：选择校准后的 TEXT。
- 两者明显分歧：根据上一可信输出选择时间连续性更好的观测；无法判定时优先 Legacy，要求 TEXT 连续确认后才切换，任一失败帧会重新计数。
- 只有一个成功：降级到成功的观测，并记录来源。
- 两者都失败：主路径立即沿用现有错误/重试路径，不把陈旧角包装成新观测；连续输入控制器由 120ms 看门狗自动减速。

Fusion 不对两个角度做无条件平均，避免把一个明显错误的观测扩散到输出。

## 连续转向控制

仅 TEXT/Fusion 使用固定 8ms 节拍的连续输入控制器：

- 小角差死区，消除正负一两度来回修正；
- 单周期最大位移和角速度限制，避免 340/1070 像素突发；
- 相邻周期最大速度增量限制，限制加速度；
- 方向翻转时先过零，阻止一帧内从大幅左转切到大幅右转；
- 停车大转向与跑动微调使用不同上限；
- 120ms 没有新控制目标时自动减速，防止识别卡顿时持续转过头。

连续输入线程只在 TEXT/Fusion 执行器内创建；`停止()` 必须先把目标速度清零、结束并回收线程，再释放移动键。Legacy 不创建该线程，继续同步调用原随机贝塞尔实现。

## 路线与近点保护

- Legacy 继续使用自动路线最小点距 18。
- TEXT/Fusion 使用更小点距 6，保留录制线路中的折点。
- Legacy 继续使用原近点保护范围与最大微调角。
- TEXT/Fusion 在距离 10 内禁止停车大转向，并把近点微调限制为较小角度。

## UI、日志和兼容性

- 三处模式入口增加 `fusion`，所有默认值保持 `legacy`。
- TEXT/Fusion 共用大雷达 ROI；Legacy ROI 不变。
- 诊断详情增加观测来源、两算法差值和控制整形后的鼠标像素，便于下一次录像对照。
- 不增加运行依赖。

## 验证标准

- 校准：标定点、跨 359°/0°、留一误差测试通过；留一 P95 小于 1.5°。
- 模式：`legacy/text/fusion` 可正规化与切换，默认仍为 `legacy`。
- Fusion：一致选择、分歧选择、单传感器降级、双失败均有测试。
- 控制：死区、限速、限加速度、反向过零、近点保护有确定性断言。
- 路线：同一输入在 Legacy 的输出与修改前一致，TEXT/Fusion 保留更多点。
- Legacy：关键函数源码/行为快照和原参数断言通过。
- 工程：`pytest`、`compileall` 通过，Git 差异不包含用户路线、日志、截图或录像。
