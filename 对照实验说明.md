# legacy vs text 对照实验说明（只分析、不改寻路）

目标：用**同一条路线、同一套巡航**，只换角度模式，量化  
「旧算法抖但能走」vs「text 准但易卡」是否由 **转向占比 / turn_wait 拒绝 / 脱困** 解释。

---

## 1. 实验条件（必须固定）

| 项 | 要求 |
|----|------|
| 路线 | **同一条** `routes/*.txt`（建议 20～40 点、含 1～2 处大折角） |
| 到点阈值 | 两边都用 **3**（或都用你常用值，但必须相同） |
| 精准模式 | 两边都关，或都开 |
| 起点 | 尽量同一出生点/同一第一点附近 |
| 分辨率/窗口 | 不变（ROI 依赖分辨率） |
| 截图 | 不改（004 默认 GDI） |
| 干扰 | 不要手动动鼠标；Esc 只用于紧急停止 |

建议路线：你刚录的 `自动录制坐标_*.txt` 或 `游客左/右` 等熟悉图。

---

## 2. 怎么跑（两轮）

### 轮 A：legacy（旧颜色轮廓）

1. 打开 `主界面.py`（或巡航脚本）
2. 角度选 **旧算法**
3. 选好路线 → **开始回放**
4. **尽量走完全程**；中途卡死超过 30s 可 Esc，并在记录表注明「未完成」
5. 记下日志文件名（见下）

### 轮 B：text（箭头）

1. **同一路线**、同样到点阈值
2. 角度选 **text 箭头**
3. 同样尽量走完全程
4. 记下日志文件名

### 日志在哪

```
三角洲录制器004\logs\巡航工具_YYYYMMDD_HHMMSS.log
```

每次开始回放会新生成一个。按时间排序，最新两个通常就是 A/B。

可选：在文件名旁自己备注：

```
巡航工具_20260723_020033.log   # legacy 游客右 完成
巡航工具_20260723_020200.log   # text 游客右 中途Esc
```

## 5. 如何解读结果（对应之前分析）

| 指标 | 若 text ≫ legacy | 含义 |
|------|------------------|------|
| **转向占比** | 明显更高 | 准角差更常触发「停车大转向」 |
| **turn_wait 次数 / 拒绝** | 更高 | 滤波滞后 → 确认环拒信/振荡 |
| **脱困次数** | 更高 | 拧头时位移≈0，触发卡住逻辑 |
| **step/s** | 更低 | 控制环大量时间花在转向确认 |
| **角差均值** | text 可能更低（更准）但仍卡 | **准不等于控得住** |
| **帧间角度跳变≥25°** | 若 text 更低却仍卡 | 问题在控制策略，不只是抖 |

**支持原结论的典型模式：**

```text
text：角差更小 + 转向占比更高 + turn_wait拒绝更多 + 脱困更多 + 时长更长/未完成
legacy：角差更大或更抖 + 转向少 + 疾跑多 + 能完成
```

**若两边指标接近、只有体感不同：**  
再查是否 UI/机器负载、是否中途手动干扰，或 text 识别失败重试（看 `识别失败`/`retry` 日志）。

---

## 6. 实验注意

1. 每模式至少 **完整 1 次**；有时间各跑 2 次取平均更稳。  
2. 不要同一局里先 text 卡死后立刻 legacy 却换了位置——尽量 **同起点**。  
3. text 若开局就识别失败，先确认游戏内雷达可见，再记「感知失败」与「控制卡死」分开。  
4. 本实验 **不改代码**；分析脚本只读日志。
